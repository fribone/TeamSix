# 项目启动书：二手书流转系统

## 项目概况

项目名称：二手书流转系统

开发模式：前后端分离

技术栈：后端 Java+Spring Boot+MyBatis，前端 Vue.js

主要受众：广州软件学院学生、校内书籍爱好者

## 背景与意义

### 现状与问题

资源浪费：广州软件学院学生手中的教材、专业参考书、课外读物等使用周期短，闲置后多被丢弃，书籍资源未得到有效利用。

交易低效：校内二手书交易多为线下零散对接，信息不对称，学生找书、售书的时间成本高，流转效率低下。

渠道缺失：缺乏校内专属的二手书流转平台，学生无统一、便捷的渠道发布和检索二手书信息，供需无法有效匹配。

### 项目意义

资源循环：搭建校内二手书专属流转渠道，实现书籍资源的循环利用，减少资源浪费，践行绿色环保理念。

降本减负：为学生提供低价购买二手书的途径，降低购书成本，减轻学生经济负担。

场景适配：打造贴合广州软件学院校园场景的轻量化平台，解决校内学生二手书交易的核心痛点，提升校园生活便捷性。

## 系统功能范围

### 用户端（前台）

|     |     |
| --- | --- |
| 登录注册 | 支持学生以学号 / 手机号完成注册登录，为所有核心功能提供权限基础。 |
| 书籍发布 | 用户可填写书籍名称、作者、分类、成色、价格等信息，支持图片上传，完成二手书信息发布。 |
| 书籍检索 | 提供书名 / 作者关键词搜索，支持按书籍分类、成色、价格区间筛选，支持按发布时间、价格排序。 |
| 书籍互动 | 用户可收藏感兴趣的书籍，可向书籍发布者留言咨询交易相关问题。 |
| 状态管理 | 发布者可对已发布书籍进行状态更新，支持待流转、已成交、已下架三种状态切换。 |
| 书籍评价 | 买家可对已成交的书籍进行星级 + 文字评价，为其他用户提供参考。 |
| 个人中心 | 统一管理我的发布（编辑 / 删除 / 改状态）、我的收藏、我的留言、评价记录等信息。 |

### 管理端（后台）

|     |     |
| --- | --- |
| 功能模块 | 功能描述 |
| 用户信息管理 | 对平台注册用户进行增删改查，查看用户发布、交易相关数据，处理违规用户。 |
| 书籍信息管理 | 对用户发布的书籍信息进行审核，下架违规书籍，对书籍分类进行维护管理。 |
| 交易信息管理 | 查看平台书籍交易记录，处理交易纠纷，统计书籍流转相关数据。 |
| 评论信息管理 | 审核用户的书籍评价内容，删除违规评论，维护平台评论环境。 |
| 数据统计模块 | 统计平台书籍发布量、成交量、分类占比等数据，生成可视化统计报表。 |

## 业务约束与非功能需求

业务规则：同一书籍仅可由发布者进行状态修改和信息编辑，已成交 / 已下架书籍不可再被检索到。

数据维护：系统需具备专业的管理后台，对书籍信息、用户评论、交易内容进行审核，确保平台信息真实合规。

核心指标：提升广州软件学院校内二手书流转效率，实现平台书籍发布量、成交量稳步增长，覆盖校内 80% 以上院系的学生需求。

使用体验：平台页面简洁、操作便捷，响应速度快，适配 PC 端和移动端，满足学生碎片化使用需求。

信息安全：保护用户的个人信息、交易留言等隐私数据，防止信息泄露和滥用。
